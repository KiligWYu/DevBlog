Hi There! 感谢阅读 Kilig 的碎周报第 140 期。本篇是对过去一周（2024.12.30 - 2025.01.05）的记录、思考与分享。



新年快乐！

本周看了 同居牢友 Let’s Go to Prison (2006)、坠落的审判 Anatomie d’une chute (2023) 和 萍水相腐檐廊下 メタモルフォーゼの縁側 (2022)。




周六发现手机上机场挂了，所有线路全部超时，备用机场所有线路也全部超时，蜂窝网络下也一样，然后发现直连模式也超时；我使用的是 Surge 这个应用，我怀疑是 DNS 的问题，但使用系统 DNS 或自定义 DNS 也还是同样的问题，想着也没改配置，前一天还正常使用呢，正一筹莫展时候，搜索发现有可能会是 DoH（DNS over HTTPS） 的问题，我又检查了下 Surge 的配置，配置文件里没有 DoH 的配置，后来发现在 Modules 下面，开启了 DoH，老早以前设置的，还开了两个，阿里的 Ali-DoH 和腾讯的 DNSPod-DoH，进一步排查发现是 DNSPod-DoH 的问题，禁用掉就好了。

DNS over HTTPS（缩写：DoH）是域名系统的安全协议，以 HTTPS 协议完成 DNS 解析来保护 网络主机的隐私，能避免传统 DNS 协议中用户的 DNS 解析请求被窃听或者修改（例如中间人攻击）的情况。—— DNS over HTTPS - Wikiwand
