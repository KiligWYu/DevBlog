本周碎片

  1. 本周开始看赫尔辛基大学的 ★ 《全栈公开课 2022》，这个公开课躺在我书签里两年了，终于开始学了 🤣，目前学到 Part 2/13。自己技术栈太单一了，所以学点前后端的东西，学完就可以做更多 side project 了。

    一站式学习 React, Redux, Node.js, MongoDB, GraphQL 以及 TypeScript！这门课程会向你介绍基于 JavaScript 的现代 Web 开发技术。重点是利用 ReactJS 搭建单页面应用程序（SPA：Single Page Application），并使用 Node.js 构建 REST API。

  2. 一点感受：对于那些需要定期清理、清理起来麻烦还清理不干净的非必需产品，买之前得慎重考虑。
  3. 我市又有新冠病例了，又天天做核酸、封城、静默管理，本月第二次了。我倒要看清零政策能坚持到什么时候。
  4. 驾照快要到期了，周一（22）下午去了车管所换了新驾照。照相、体检、填表，本来接下来就是制证，可以很快搞定的，但因为我的驾照是外地的，得去大厅窗口办理，排队一小时后，几分钟办理完毕。我也是老司机了，六年驾龄，不过里程倒是很短 😄。
  5. #产品动态 这段时间 side projects 就不发新版了，也发不了，电脑都升级成测试版系统了，都正式版系统发布，第一时间发新版。

  2. ★ 李辉 - 待业五年后的找工作经历
    博主很厉害了，定位清晰、目标明确、有的放矢，详细介绍了他的外企面试经历。摘录文中的一段：

    我对目标职位的三个需求是：

    1. 不加班
    2. 面试不考算法题
    3. 现场（on-site）工作

    「不加班」是基本要求，我需要一份正常工时的工作，一来是为身心健康着想，二来是因为我需要有业余时间来写书和做开源项目。

    「面试不考算法题」是因为不想浪费时间去刷无聊的算法题。算法题对我来说没有任何乐趣（换句话说，太难了），相比之下，我更喜欢开发 Web 程序和设计 API。

    想要一个「现场（on-site）工作」是因为很想去体验办公室生活，顺便体验在不同的城市生活。在家呆了五年后，我也有些厌倦在家工作。太容易过度娱乐和过度工作，模糊生活工作的边界。我想要有一个办公空间来把工作和生活在时间和空间上分隔起来。

    国内公司普遍加班，互联网公司普遍考算法题，因此能够满足这三点的只有「非互联网 + 外企」职位。结合我擅长且感兴趣的技术栈（Python Web）和想要去的城市（上海），最终的求职方向是「上海 | 外企 | Python 后端工程师」。

    我目前待业在家，下次要找工作，需求是：双休、不加班、面试不考算法题、按时发工资。郑州的就业环境真的是 😑，自己又菜，按这要求找就很难了。
